# Правила работы с репозиторием

1. Перед изменениями изучать `README.md`, `AGENTS.md`, `docs/GAME_DESIGN.md`, `docs/ARCHITECTURE.md` и затрагиваемые файлы.
2. Не переписывать весь проект ради небольшой задачи.
3. Не удалять работающие функции без прямого указания.
4. Вносить минимальные и понятные изменения.
5. Не использовать тяжёлые фреймворки, сборщики, npm и внешние зависимости без явного разрешения.
6. Проект должен запускаться как обычный статический сайт на GitHub Pages.
7. Использовать HTML, CSS и чистый JavaScript.
8. Не хранить секреты, токены, пароли и персональные данные в репозитории.
9. Не менять файлы вне текущего репозитория.
10. Перед завершением проверять синтаксис изменённых JavaScript-файлов и отсутствие очевидных ошибок.
11. Сохранять поддержку мобильных экранов.
12. Не создавать огромные файлы: разделять код по назначению, когда это действительно нужно.
13. Игровое состояние хранить отдельно от отображения интерфейса.
14. Не смешивать в одном месте игровую логику, работу с DOM и данные.
15. При изменении архитектуры обновлять `docs/ARCHITECTURE.md`.
16. При добавлении или изменении игровой механики обновлять `docs/GAME_DESIGN.md`.
17. После каждой завершённой задачи добавлять краткую запись в `docs/CHANGELOG.md`.
18. В итоговом ответе всегда указывать:
    - что изменено;
    - какие файлы затронуты;
    - как проверить результат;
    - какие ограничения или нерешённые вопросы остались.
19. Не делать commit и push самостоятельно: этим занимается Telegram-мост.
20. Если запрос неоднозначен и есть риск существенно изменить проект не так, сначала описать безопасное предположение или задать уточняющий вопрос, а не придумывать крупные решения.
