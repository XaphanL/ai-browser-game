# Архитектура проекта

## Текущее состояние

Проект является статическим сайтом без сборки и внешних зависимостей. Рабочая страница находится в корневом `index.html`: сейчас она содержит HTML-разметку и встроенные CSS-стили. JavaScript и игровые механики пока отсутствуют.

Текущая структура:

```text
.
├── index.html              # существующая автономная страница
├── README.md               # описание и инструкции по запуску
├── AGENTS.md               # правила работы с репозиторием
├── css/                    # будущие таблицы стилей
├── js/
│   └── game/               # будущие отдельные игровые механики
├── assets/
│   ├── images/             # будущие изображения
│   └── audio/              # будущие аудиоматериалы
├── docs/                   # игровая и проектная документация
└── tests/                  # будущие проверки
```

В корне также сохранены существующие служебные и проверочные файлы `TEST.md`, `github_push_test.txt` и `hello.txt`. Они не участвуют в работе страницы.

Пустые каталоги отмечены файлами `.gitkeep`, чтобы Git сохранял структуру. Эти файлы не являются исходным кодом.

## Рекомендуемое разделение при развитии проекта

Когда появится игровой код и его объём потребует разделения, рекомендуется использовать следующую ответственность модулей:

- `main.js` — запуск приложения и связывание модулей;
- `state.js` — состояние игры и операции над ним;
- `ui.js` — отображение и обработка интерфейса;
- `js/game/` — отдельные игровые механики.

Это ориентир для дальнейшей разработки, а не описание уже реализованных систем и не требование немедленно переносить существующий код.
