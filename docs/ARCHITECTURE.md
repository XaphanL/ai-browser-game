# Архитектура проекта

## Текущее состояние

Проект является статическим сайтом без сборки и внешних зависимостей. Рабочая страница находится в корневом `index.html`; стили и JavaScript подключаются отдельными файлами.

Текущая структура:

```text
.
├── index.html              # разметка единственного игрового экрана
├── css/
│   └── styles.css          # оформление и адаптивная раскладка
├── js/
│   ├── main.js             # запуск игры и связывание действий с интерфейсом
│   ├── ui.js               # отображение состояния и обработчики интерфейса
│   └── game/
│       ├── data.js         # баланс, маршрут и описания модулей
│       └── state.js        # состояние и чистая игровая логика
├── README.md               # описание и инструкции по запуску
├── AGENTS.md               # правила работы с репозиторием
├── assets/
│   ├── images/             # будущие изображения
│   └── audio/              # будущие аудиоматериалы
├── docs/                   # игровая и проектная документация
└── tests/                  # будущие проверки
```

В корне также сохранены существующие служебные и проверочные файлы `TEST.md`, `github_push_test.txt` и `hello.txt`. Они не участвуют в работе страницы.

Пустые каталоги отмечены файлами `.gitkeep`, чтобы Git сохранял структуру. Эти файлы не являются исходным кодом.

## Разделение ответственности

Игровые данные и числовой баланс находятся в `js/game/data.js`. Изменения состояния выполняет `js/game/state.js`, не обращаясь к DOM. Модуль `js/ui.js` строит интерфейс из текущего состояния, а `js/main.js` связывает эти части и отвечает за перезапуск сессии.

Состояние хранится только в памяти вкладки. Сохранение между сессиями в минимальной версии не предусмотрено.
