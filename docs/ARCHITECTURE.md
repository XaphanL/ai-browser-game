# Архитектура проекта

## Текущее состояние

Проект является статической браузерной игрой без сборки и внешних зависимостей. `index.html` содержит каркас интерфейса, стили вынесены в `css/styles.css`, а игра реализована ES-модулями на чистом JavaScript и Canvas 2D.

Текущая структура:

```text
.
├── index.html              # каркас игрового экрана и мобильных контролов
├── css/styles.css          # интерфейс, адаптивность и экранные контролы
├── js/
│   ├── main.js             # запуск цикла и обработка игровых событий
│   ├── config.js           # параметры баланса игрока, комнат и босса
│   ├── maze.js             # генерация графа лабиринта и история переходов
│   ├── upgrades.js         # каталог улучшений, состояние забега и операции магазина
│   ├── state.js            # создание игрового состояния
│   ├── game.js             # игровая симуляция без работы с DOM
│   ├── input.js            # клавиатура, мышь и сенсорные кнопки
│   ├── renderer.js         # отрисовка состояния на Canvas
│   ├── minimap.js          # отдельная отрисовка посещённого маршрута
│   └── ui.js               # обновление HTML-интерфейса
├── README.md               # описание и инструкции по запуску
├── AGENTS.md               # правила работы с репозиторием
├── css/                    # будущие таблицы стилей
├── assets/
│   ├── images/             # будущие изображения
│   └── audio/              # будущие аудиоматериалы
├── docs/                   # игровая и проектная документация
└── tests/                  # будущие проверки
```

В корне также сохранены существующие служебные и проверочные файлы `TEST.md`, `github_push_test.txt` и `hello.txt`. Они не участвуют в работе страницы.

Пустые каталоги отмечены файлами `.gitkeep`, чтобы Git сохранял структуру. Эти файлы не являются исходным кодом.

## Разделение ответственности

`maze.js` строит связный граф комнат и хранит операции истории маршрута. `state.js` создаёт обычный объект состояния конкретной комнаты и не отвечает за отображение. Тип комнаты (`arena`, `merchant` или `boss`), ячейки и заряд восьми граней брони и параметры противников хранятся в игровом состоянии. Долгоживущее состояние забега — граф лабиринта, посещённые комнаты, пройденные связи, кэш состояний комнат, очки, улучшения, активный взаимоисключающий модуль и итоговые характеристики — создаётся в `upgrades.js`. `game.js` изменяет только активную комнату по входным данным, исполняет способности модулей, начисляет награды и возвращает события переходов. Неактивные комнаты остаются в кэше без симуляции, поэтому при возврате восстанавливаются в состоянии на момент выхода. `renderer.js` рисует арену, `minimap.js` — известную часть маршрута, а `ui.js` обновляет HUD. `main.js` связывает компоненты, переносит состояние игрока между комнатами, управляет повторными покупками и открывает отдельный ассортимент торговца при первом входе в его комнату.
